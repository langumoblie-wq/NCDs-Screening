# 🩺 NCDs-Screening (ระบบคัดกรองเบาหวาน-ความดันโลหิตสูง)

ระบบเว็บแอปพลิเคชันคัดกรองโรคไม่ติดต่อเรื้อรัง (NCDs) เบาหวานและความดันโลหิตสูง พร้อมระบบประเมินความพฤติกรรมการบริโภคอาหาร (หวาน มัน เค็ม) และการวิเคราะห์กลุ่มเสี่ยงด้วยปิงปอง 7 สี

---

## 🛠️ คู่มือการตั้งค่าเพื่อใช้งานจริง (Supabase + GitHub Pages)

เพื่อให้แอปพลิเคชันสามารถ **บันทึก อ่าน แก้ไข และลบข้อมูลได้จริงบนอินเทอร์เน็ต** และใช้งานผ่าน GitHub Pages ได้อย่างสมบูรณ์แบบ โปรดทำตามขั้นตอนเหล่านี้:

### ขั้นตอนที่ 1: การตั้งค่าฐานข้อมูล Supabase
1. ไปที่เว็บไซต์ [https://supabase.com/](https://supabase.com/) และสมัครสมาชิก (ฟรี)
2. กดปุ่ม **New Project** เพื่อสร้างโปรเจกต์ใหม่ (ตั้งชื่อและรหัสผ่านตามต้องการ)
3. รอจนกว่าระบบจะสร้างโปรเจกต์เสร็จสมบูรณ์ (ใช้เวลาประมาณ 1-2 นาที)
4. ไปที่เมนู **SQL Editor** (ไอคอนรูป <code>&lt;/&gt;</code> ที่แถบด้านซ้าย)
5. เปิดไฟล์ `supabase_setup.sql` ที่มีอยู่ในโปรเจกต์นี้ คัดลอกโค้ดทั้งหมด ไปวางใน SQL Editor ของ Supabase แล้วกด **RUN** (เพื่อสร้างตารางเก็บข้อมูลและกำหนดสิทธิ์)

### ขั้นตอนที่ 2: การคัดลอกค่าเชื่อมต่อ (API Keys)
1. ในหน้า Supabase ให้ไปที่เมนู **Settings** (รูปฟันเฟืองล่างซ้าย) -> เลือก **API**
2. หาหัวข้อ **Project URL** และคัดลอก URL ไว้
3. หาหัวข้อ **Project API keys** (ตัวที่เขียนว่า `anon` / `public`) และคัดลอกค่านี้ไว้

### ขั้นตอนที่ 3: การตั้งค่า GitHub Secrets (เพื่อให้เว็บรู้จักฐานข้อมูล)
1. เปิดหน้า Repository ของท่านใน GitHub (ลิงก์: https://github.com/langumoblie-wq/NCDs-Screening)
2. ไปที่เมนู **Settings** (รูปฟันเฟืองขวาบนของ Repository)
3. เลื่อนแถบด้านซ้ายมือลงมาที่หัวข้อ **Security** -> เลือก **Secrets and variables** -> เลือก **Actions**
4. กดปุ่ม **New repository secret** เพื่อสร้างตัวแปร 2 ตัว ดังนี้:
   - **ตัวที่ 1**: 
     - Name: `VITE_SUPABASE_URL`
     - Secret: `(วาง URL ที่คัดลอกมาจากขั้นตอนที่ 2)`
   - **ตัวที่ 2**: 
     - Name: `VITE_SUPABASE_ANON_KEY`
     - Secret: `(วาง anon key ที่คัดลอกมาจากขั้นตอนที่ 2)`

### ขั้นตอนที่ 4: การเปิดใช้งานระบบ GitHub Pages อัตโนมัติ
หากคุณเจอปัญหาหน้าเว็บขาว หรือ Action Error ตอน Deploy ให้ตั้งค่าดังนี้:
1. ในหน้า GitHub Repository ของคุณ ไปที่เมนู **Settings**
2. เมนูด้านซ้าย เลือก **Pages**
3. ในหัวข้อ **Build and deployment** -> ตรง **Source** ให้กดเปลี่ยนเป็น **`GitHub Actions`** (สำคัญมาก*)
4. แนะนำให้ตั้งค่า **Read & Write permissions** ของ Actions ด้วย: 
   - ไปที่ Settings -> Actions -> General
   - เลื่อนลงมาที่หัวข้อ **Workflow permissions** 
   - เลือก **Read and write permissions** แล้วกด Save
5. หลังจากตั้งค่าทั้งหมดเสร็จสิ้น ให้แก้ไขไฟล์ใดก็ได้ 1 ไฟล์ (เช่น เติมเคาะวรรค 1 ทีในไฟล์ README.md) แล้วทำการ **Commit & Push** โค้ดไปยัง GitHub ระบบจะทำการดึงข้อมูลจาก Supabase และสร้างหน้าเว็บที่ใช้งานได้สมบูรณ์ทันที

## 🌟 ฟังก์ชันและการอัปเดตล่าสุด
1. **โค้ดฐานข้อมูลสมบูรณ์แล้ว**: ตัวเว็บมีการเชื่อมโค้ดแบบ `upsert`, `select`, และ `delete` ใช้งานกับตาราง `ncd_records` ได้อย่างไร้รอยต่อ
2. **ระบบป้องกันจอขาว (Null/Undefined Safety)**: หากข้อมูลบางตัวในฐานข้อมูลแหว่งหายไป ระบบจะไม่มีอาการจอขาว
3. **ระบบออฟไลน์ชั่วคราว (Offline Backup)**: หากเน็ตหลุดชั่วคราว ระบบจะดึงข้อมูลจาก `localStorage` มาสำรองการทำงานไว้ก่อนจนกว่าจะต่อ Supabase ติด
